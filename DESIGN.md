# Design Direction

## Surface

Browser Fidelity Lab is a repeated-use developer and QA tool. The first screen is
the scan workspace, not a marketing landing page.

## References

- Linear: compact information density, low chrome, calm hierarchy.
- Stripe docs/tools: crisp labels, strong tables, useful empty states.
- PostHog product surfaces: product-led technical copy and clear status colors.

## Rules

- Background: quiet neutral with white work surfaces and ink text.
- Accent: teal for action and score emphasis; amber/red/green only for states.
- Radius: 8px or less.
- Cards are only panels for tools, reports, or repeated list items.
- No hero gradient, decorative blobs, three-card feature routine, or generic
  marketing copy.
- Buttons use icon plus text where action clarity matters.
- Sensitive tests require explicit user action before collecting signals.

## Visual QA

Validate desktop and mobile first viewport, table overflow, button text, focus
states, and report readability in browser QA.
